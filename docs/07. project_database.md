# EgoFlow Server Database

이 문서는 현재 `ego-flow-server`의 데이터베이스 구조를 정리한 문서다. Prisma schema 기준으로 enum, 테이블, 인덱스, 논리적 연관 관계를 설명한다.

## 1. 사용 기술

- DBMS: PostgreSQL
- ORM/Schema: Prisma
- migration 관리: `backend/prisma/migrations/`

## 2. Enum

### 2.1 `UserRole`

| 값 | 의미 |
| --- | --- |
| `admin` | 시스템 관리자 |
| `user` | 일반 사용자 |

### 2.2 `RepoVisibility`

| 값 | 의미 |
| --- | --- |
| `public` | membership이 없어도 read 가능 |
| `private` | member 또는 system admin만 접근 가능 |

### 2.3 `RepoRole`

| 값 | 의미 |
| --- | --- |
| `read` | 조회/재생 권한 |
| `maintain` | 운영 권한 |
| `admin` | repository 관리 권한 |

### 2.4 `VideoStatus`

| 값 | 의미 |
| --- | --- |
| `PENDING` | 대기 |
| `PROCESSING` | 처리 중 |
| `COMPLETED` | 완료 |
| `FAILED` | 실패 |

### 2.5 `RecordingSessionStatus`

| 값 | 의미 |
| --- | --- |
| `PENDING` | register 완료, stream ready 전 |
| `STREAMING` | 송출 중 |
| `STOP_REQUESTED` | 사용자 종료 의도 수신 |
| `FINALIZING` | 실제 송출 종료 후 flush 대기 |
| `COMPLETED` | 최종 video 생성 완료 |
| `FAILED` | finalize 실패 |
| `ABORTED` | publish로 이어지지 못함 |

### 2.6 `RecordingSessionEndReason`

| 값 | 의미 |
| --- | --- |
| `USER_STOP` | 앱에서 stop 요청 |
| `GLASSES_STOP` | 안경 측 촬영 종료 |
| `UNEXPECTED_DISCONNECT` | 예기치 않은 송출 종료 |
| `REGISTRATION_TIMEOUT` | register 후 publish 미시작 |
| `INTERNAL_ERROR` | 내부 처리 오류 |

### 2.7 `RecordingSegmentStatus`

| 값 | 의미 |
| --- | --- |
| `WRITING` | segment 작성 중 |
| `COMPLETED` | segment 작성 완료 |
| `FAILED` | segment 처리 실패 |

## 4. 테이블 상세

### 4.1 `users`

계정 테이블이다.

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | `varchar(64)` PK | 사용자 ID |
| `password_hash` | `varchar(255)` | bcrypt hash |
| `role` | `UserRole` | 시스템 역할 |
| `is_active` | `boolean` | 활성 상태 |
| `display_name` | `varchar(255)` nullable | 표시 이름 |
| `created_at` | `timestamp` | 생성 시각 |
| `updated_at` | `timestamp` | 수정 시각 |

### 4.2 `settings`

서버 설정 key-value 저장소다.

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `key` | `varchar(255)` PK | 설정 키 |
| `value` | `text` | 설정 값 |
| `updated_at` | `timestamp` | 수정 시각 |

### 4.3 `repositories`

repository 메타데이터 테이블이다.

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | `uuid` PK | repository ID |
| `name` | `varchar(64)` | repository 이름, RTMP path name으로도 사용 |
| `owner_id` | `varchar(64)` | repository owner user id |
| `visibility` | `RepoVisibility` | 공개 범위 |
| `description` | `varchar(500)` nullable | 설명 |
| `created_at` | `timestamp` | 생성 시각 |
| `updated_at` | `timestamp` | 수정 시각 |

제약 및 인덱스:

- unique: `(owner_id, name)`
- index: `owner_id`
- index: `visibility`

### 4.4 `repo_members`

repository별 사용자 권한 매핑 테이블이다.

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | `uuid` PK | row ID |
| `repository_id` | `uuid` | 대상 repository |
| `user_id` | `varchar(64)` | 대상 user |
| `role` | `RepoRole` | repository role |
| `created_at` | `timestamp` | 생성 시각 |

제약 및 인덱스:

- unique: `(repository_id, user_id)`
- index: `repository_id`
- index: `user_id`

### 4.5 `recording_sessions`

사용자 1회 촬영을 나타내는 핵심 테이블이다.

| 컬럼 | 타입 | 설명 |

| `not_ready_at` | `timestamp` nullable | stream-not-ready 수신 시각 |

## 5. 논리적 관계

### 5.1 User ↔ Repository

직접 relation 필드는 없지만 논리적으로 다음 두 경로가 있다.

- `repositories.owner_id = users.id`
- `repo_members.user_id = users.id`

### 5.2 Repository ↔ RecordingSession

- `recording_sessions.repository_id = repositories.id`

즉 recording은 항상 하나의 repository에 속한다.

### 5.3 RecordingSession ↔ RecordingSegment

- `recording_segments.recording_session_id = recording_sessions.id`

즉 recording session 하나는 여러 raw segment를 가질 수 있다.

### 5.4 RecordingSession ↔ Video

- `videos.recording_session_id = recording_sessions.id`

현재 설계의 핵심은 `RecordingSession 1개 -> Video 1개`다.

### 5.5 Repository ↔ Video

- `videos.repository_id = repositories.id`

video는 항상 하나의 repository에 속한다.

## 6. 현재 구현에서 중요한 데이터 규칙

- repository는 스트리밍, 저장, 조회, 권한의 기준 단위다
- recording의 lifecycle source of truth는 PostgreSQL이다
- active live pointer는 Redis에 저장되지만 terminal state는 DB에서 판단한다
- final video 생성 단위는 segment가 아니라 recording 전체다
- generated file 경로는 DB에 절대경로로 저장된다
- `settings.target_directory`는 부팅 시 실제 환경 변수와 동기화된다

## 7. 현재 미사용 또는 확장 예정 성격의 필드

`videos`의 아래 필드는 schema에는 있으나 현재 worker가 채우지 않는다.

- `clip_segments`
- `action_labels`
- `video_text_alignment`
- `scene_summary`
