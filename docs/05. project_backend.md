# EgoFlow Server Backend

이 문서는 현재 `ego-flow-server/backend` 패키지의 구조와 책임을 정리한 문서다.

## 1. 역할

backend는 아래 책임을 가진다.

- REST API 제공
- JWT 인증 및 repository 권한 검사
- MediaMTX RTMP/HLS 인증 처리
- MediaMTX ready/not-ready/segment hook 처리
- generated file 접근 제어
- Prisma/Redis/BullMQ 연결 관리

## 2. 디렉토리 구조

```text
backend/
├── prisma/
│   ├── schema.prisma
│   ├── migrations/
│   ├── manual-indexes.sql
│   └── seed.ts
└── src/
    ├── index.ts
    ├── worker.ts
    ├── config/
    ├── lib/
    ├── middleware/
    ├── routes/
    ├── schemas/
    ├── services/
    ├── types/
    └── workers/
```

## 3. 주요 엔트리포인트

| 파일 | 역할 |
| --- | --- |
| `src/index.ts` | Express 앱 부팅, route/middleware 등록 |
| `src/worker.ts` | BullMQ worker 프로세스 부팅 |
| `prisma/seed.ts` | admin 계정 및 setting bootstrap |

## 4. 레이어 구조

### 4.1 routes

HTTP endpoint를 정의하고, validation/middleware/service 호출을 연결한다.

| 파일 | 역할 |
| --- | --- |
| `auth.routes.ts` | 로그인, MediaMTX 인증 |
| `admin.routes.ts` | 관리자 API |
| `repositories.routes.ts` | repository 및 member 관리 |
| `streams.routes.ts` | stream 등록, active stream 조회 |
| `recordings.routes.ts` | recording 상태 조회, stop |
| `hooks.routes.ts` | MediaMTX ready/not-ready/segment hook |
| `users.routes.ts` | 본인 비밀번호 변경 |
| `videos.routes.ts` | video 목록/상세/상태/삭제 |

### 4.2 middleware

| 파일 | 역할 |
| --- | --- |
| `auth.middleware.ts` | JWT 검증, 사용자 재확인, 토큰 갱신 헤더 |
| `role.middleware.ts` | system role 검사 |
| `repo-access.middleware.ts` | repository 최소 권한 검사 |
| `file-access.middleware.ts` | `/files/*` 경로 접근 제어 |
| `validate.middleware.ts` | Zod request validation |
| `error.middleware.ts` | 공통 에러 응답 |

### 4.3 services

도메인 로직이 집중된 계층이다.

| 파일 | 역할 |
| --- | --- |
| `auth.service.ts` | 로그인, RTMP 인증, 비밀번호 변경 |
| `admin.service.ts` | 사용자 관리, setting 조회 |
| `repository.service.ts` | repository CRUD, 권한 해석, member 관리 |
| `stream.service.ts` | register/active stream 조회, reconcile loop |
| `recording-session.service.ts` | recording lifecycle, Redis live pointer, hook 처리 |
| `video.service.ts` | video 조회/삭제 |
| `processing.service.ts` | BullMQ enqueue 및 progress 조회 |

### 4.4 lib

| 파일 | 역할 |
| --- | --- |
| `prisma.ts` | Prisma singleton |
| `redis.ts` | Redis singleton |
| `jwt.ts` | access token sign/verify/refresh 판단 |
| `storage.ts` | target directory 초기화, 파일 URL 변환 |
| `bullmq.ts` | BullMQ connection 생성 |
| `ffprobe.ts` | 영상 메타데이터 추출 |
| `errors.ts` | 공통 AppError |

### 4.5 workers

| 파일 | 역할 |
| --- | --- |
| `recording-finalize.worker.ts` | recording 기반 finalize worker |
| `encoding.ts` | output path 계산, ffmpeg 인코딩 helper |

## 5. 요청 처리 흐름

대부분의 요청은 다음 패턴을 따른다.

```text
route
→ auth / role / repo-access / validation middleware
→ service
→ prisma / redis / storage / queue
→ response
```

## 6. backend가 직접 관리하는 외부 시스템

- PostgreSQL
- Redis
- BullMQ
- MediaMTX auth/webhook/API 연동
- 로컬 파일 시스템의 generated dataset 경로

## 7. 구현상 특징

- 모든 주요 입력은 Zod schema로 검증한다.
- repository 권한 해석은 `repository.service.ts`에 집중되어 있다.
- `/files/*`도 일반 정적 파일 서빙이 아니라 인증과 repository 접근 제어를 거친다.
- worker는 별도 패키지가 아니라 backend 패키지 내부 엔트리로 분리되어 있다.
