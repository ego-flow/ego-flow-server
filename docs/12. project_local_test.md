# EgoFlow Server Local Real-Device Test

이 문서는 현재 EgoFlow의 로컬 실기기 테스트 절차를 정리한 문서다. 기준 시나리오는 다음과 같다.

- host PC에서 `ego-flow-server`를 Docker Compose로 실행
- Android 실기기를 `adb`로 연결
- app은 `adb reverse`를 통해 host의 local server로 접속

상세 구현 설명은 기존 문서를 참고한다.

- runtime/deploy: [03. project_runtime.md](./03.%20project_runtime.md), [11. project_deploy.md](./11.%20project_deploy.md)
- app RTMP path: [/home/dennis0405/ego-flow/ego-flow-app/docs/04. project_rtmp_android.md](/home/dennis0405/ego-flow/ego-flow-app/docs/04.%20project_rtmp_android.md)
- workspace 운영 메모: [/home/dennis0405/ego-flow/AGENTS.md](/home/dennis0405/ego-flow/AGENTS.md)

## 1. 현재 테스트 기준

- host server의 public HTTP 포트는 `80` 유지
- 실기기에서는 `adb reverse tcp:8080 tcp:80` 사용
- 따라서 app의 backend base URL은 `http://127.0.0.1:8080`
- RTMP/RTMPS/HLS는 필요 시 각각 같은 포트 번호로 `adb reverse`
- 서버 배포/재기동은 항상 `ego-flow-server/scripts/run.sh` 기준

핵심 포트:

- HTTP proxy: host `80`, device `8080 -> 80`
- RTMP ingest: `1935`
- RTMPS ingest: `1936`
- HLS output: `8888`

## 2. 사전 준비

### 2.1 실기기 연결 확인

```bash
adb devices
adb reverse --list
```

### 2.2 adb reverse 설정

```bash
adb reverse tcp:8080 tcp:80
adb reverse tcp:1935 tcp:1935
adb reverse tcp:1936 tcp:1936
adb reverse tcp:8888 tcp:8888
```

### 2.3 server 설정 확인

현재 local baseline 예시:

```json
{
  "TARGET_DIRECTORY": "~/ego-flow/ego-flow-data",
  "PUBLIC_HTTP_PORT": 80,
  "RTMP_PORT": 1935,
  "RTMPS_PORT": 1936,
  "HLS_PORT": 8888
}
```

확인할 값:

- `TARGET_DIRECTORY`
- `PUBLIC_HTTP_PORT=80`
- `RTMP_PORT`
- `RTMPS_PORT`
- `HLS_PORT`

`.env`에서는 아래 값도 같이 확인한다.

- `PUBLIC_RTMP_BASE_URL`
- `MEDIAMTX_API_URL`

### 2.4 app 준비

실기기 테스트에서 `CameraAccessAndroid`를 사용한다.

```bash
cd /home/dennis0405/ego-flow/ego-flow-app/samples/CameraAccessAndroid
./gradlew assembleDebug
./gradlew installDebug
adb shell am start -n com.meta.wearable.dat.externalsampleapps.cameraaccess/.MainActivity
```

app 설정 기준:

- backend base URL: `http://127.0.0.1:8080`
- repository/user/password 등 EgoFlow backend 접속 정보
- 필요 시 RTMP debug overlay 활성화

## 5. 주의 사항

- 테스트 종료나 설정 변경 전에는 `./scripts/run.sh down`으로 stack을 내린 뒤 진행한다.
- `adb reverse tcp:80 tcp:80`은 현재 실기기 기준으로 사용하지 않는다.
- app 설정은 `http://127.0.0.1:8080`, host server는 `PUBLIC_HTTP_PORT=80`이라는 점을 혼동하지 않아야 한다.
- `./scripts/run.sh reset`은 파괴적 명령이므로 일반 테스트 준비 절차에는 포함하지 않는다.

5. health/API/dashboard 접근 확인
6. 실기기에서 login / repository 선택 / 촬영 또는 스트리밍 테스트 시작
