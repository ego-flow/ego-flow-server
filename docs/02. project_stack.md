# EgoFlow Server Stack

`ego-flow-server`는 backend API, background worker, dashboard UI, MediaMTX 기반 스트리밍 인프라를 하나의 Compose 스택으로 묶은 Docker-first 프로젝트다.

## 1. 기술 스택

### 1.1 Runtime / Infra

| 영역 | 구현 | 비고 |
| --- | --- | --- |
| API Server | Express 5, TypeScript, Node.js | `backend/` |
| Dashboard | TanStack Start, React 19, Vite | `frontend/` |
| Database | PostgreSQL 16 | Prisma schema/migration 사용 |
| Cache / Queue Backend | Redis 7 | stream session cache + BullMQ backend |
| Job Queue | BullMQ 5 | 영상 후처리 queue |
| Streaming / HLS | MediaMTX | RTMP ingest, HLS remux, recording hook, HTTP auth |
| ORM | Prisma 6 | PostgreSQL schema, migration, seed |
| Validation | Zod 4 | env, request payload validation |
| Auth | JWT (HS256), bcryptjs | access token 단일 구조 |
| Media Processing | ffmpeg-static, ffprobe-static, fluent-ffmpeg | 메타데이터 추출 + 인코딩 |
| Styling | Tailwind CSS 4 | dashboard UI |

### 1.2 Backend package 주요 의존성

| 분류 | 패키지 |
| --- | --- |
| Web/API | `express`, `cors`, `helmet`, `morgan` |
| Types/Build | `typescript`, `ts-node`, `nodemon`, `@types/*` |
| DB | `prisma`, `@prisma/client` |
| Auth | `jsonwebtoken`, `bcryptjs` |
| Queue/Cache | `bullmq`, `ioredis` |
| Media | `fluent-ffmpeg`, `ffmpeg-static`, `ffprobe-static` |
| Validation | `zod` |

### 1.3 Frontend package 주요 의존성

| 분류 | 패키지 |
| --- | --- |
| App Framework | `@tanstack/react-start`, `@tanstack/react-router`, `@tanstack/react-query` |
| UI | `react`, `react-dom`, `lucide-react`, `radix-ui` |
| HTTP / Data | `axios`, `zod` |
| Playback | `hls.js` |
| Styling | `tailwindcss`, `@tailwindcss/vite`, `tailwind-merge`, `class-variance-authority` |
| Tooling | `vite`, `biome`, `vitest` |
